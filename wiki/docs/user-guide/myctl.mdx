MyCTL is a utility for controlling various parts of MyDE from the terminal or a script. It's essentially an abstraction layer for various jobs.

## Core Concepts

A `myctl` query is organized into a hierarchical command structure. The basic syntax is:

```shell
myctl <command> <subcommand> [arguments...]
```

1.  **`command`**: A top-level category of actions (e.g., `get`, `set`, `show`).
2.  **`subcommand`**: A specific action within that category (e.g., `theme`, `powermenu`).
3.  **`arguments`**: Any additional values required by the subcommand.

- To see all available commands, run `myctl help`.
- To see the subcommands for a specific command, run `myctl <command> help`.

## Available Commands

Below is a list of available commands and their usage.


## 1. get

Retrieve info about the current system configuration.

Usage: `myctl get <subcommand> [arguments...]`

### `get desktop-file`

Finds the full path of an application's `.desktop` file.

Usage: `myctl get desktop-file <application_name>`

**Example:**
```shell
myctl get desktop-file code
```

---

### `get desktop-filename`

Similar to `desktop-file`, but returns only the filename.

Usage: `myctl get desktop-filename <application_name>`

**Example:**
```shell
myctl get desktop-filename code
```

## 2. set

Usage: `myctl set <subcommand> [arguments...]`

<Tabs groupId="set-theme">

**Example:**

</TabItem>
<TabItem value="rofi" label="rofi">

Usage: `myctl set theme rofi "<theme_name>"`

**Example:**
```shell
myctl set theme rofi "catppuccin-mocha"
```

</TabItem>
</Tabs>

---

### `set volume`

Modifies the speaker (sink) volume or mute status.

<Tabs groupId="set-volume">

- Set volume to 75%: `myctl set volume 75`
- Increase volume by 10%: `myctl set volume +10`
- Decrease volume by 10%: `myctl set volume -10`

## 3. show

The `show` command displays various utility menus and dialogs, typically using Rofi.

Usage: `myctl show <subcommand>`

### `show powermenu`

Displays an interactive power menu.

Usage: `myctl show powermenu`

---

### `show keybinds`

Displays a searchable list of the keybindings defined in the MyDE documentation. This is a quick way to reference shortcuts without opening the full documentation.

Usage: `myctl show keybinds`

---

### `show tui`

Opens a terminal-based application in a new floating terminal window. This is useful for running quick tasks without interrupting your workflow.

Usage: `myctl show tui [options] [command]`

**Arguments:**

| Argument | Description |
| :--- | :--- |
| `-t`, `--term` | Terminal emulator to use (default: `wezterm start`). |
| `-c`, `--class` | Application class for the terminal window. |
| `-e`, `--exec` | Command to run. |
| `command` | The command to run in the terminal. |

**Examples:**

Run `btop` in a new terminal window:
```shell
myctl show tui btop
```

Run `lazygit` in a `kitty` terminal with a specific class:
```shell
myctl show tui -t kitty -c lazygit lazygit
```

## 4. reload

The `reload` command reloads MyDE services. Currently, it reloads the Hyprland configuration.

Usage: `myctl reload`
